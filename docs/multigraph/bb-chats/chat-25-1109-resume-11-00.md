## Контекст
- После автоматизации интеграционных тестов локально необходим запуск в CI, чтобы ловить регрессии по PR/commit.
- Требовалось использовать имеющийся `pytest` набор и предусмотреть работу с секретом подключения к живой базе.

## Что сделано
1. **GitHub Actions**
   - Добавлен workflow `.github/workflows/integration-tests.yml` (push/pull_request), который:
     - устанавливает Python 3.12;
     - ставит `requirements-dev.txt`;
     - запускает `pytest -m integration` из каталога `mgsrc/backend`.
   - Джоб выполняется только при наличии секрета `FEDOC_DATABASE_URL`, чтобы не падать в пустых окружениях.
2. **Документация**
   - README backend дополнен разделом «CI» с пояснением про секрет и порядок запуска.

## Итог
